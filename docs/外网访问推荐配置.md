# 外网访问推荐配置

Lan Control Hub 不建议把控制端口直接映射到公网。推荐做法是让每台设备加入同一个虚拟专网，再用应用内的手动连接和设备信任完成配对。

## 推荐方案：每台设备安装 Tailscale

1. 每台电脑安装 Tailscale，并登录同一个 tailnet。
2. 在 Tailscale 管理界面或客户端里复制对方设备的 `100.x.x.x` 地址。
3. 打开 Lan Control Hub，确认两台设备使用同一个加入密钥。
4. 在设置页的“网络”页签输入对方地址，例如 `100.x.x.x` 或 `100.x.x.x:46882`。
5. 连接成功后，双方在“信任”页签里点击信任对方设备。

这种方式不需要“跳一台设备”。只要每台设备都装了 Tailscale，它们会像在同一个安全内网里一样互通。

## WebRTC STUN/TURN

实时看屏和远程控制使用 WebRTC。默认不配置 ICE 服务器，优先走局域网或虚拟专网直连。

如果两台设备已经能通过 Tailscale、ZeroTier 或 WireGuard 互相访问，但实时流偶尔无法建立，可以在设置页“网络 / WebRTC 连接”里添加 STUN 或 TURN 服务器：

```text
stun:stun.example.com:3478
turn:turn.example.com:3478
```

TURN 用户名和凭据只保存在本机，不会通过设备发现广播。只有明确要强制走 TURN 中继时，才把策略切到“仅中继”。

## 高级方案：Subnet Router

如果有些设备不能安装 Tailscale，可以选择一台一直在线的电脑或小主机作为 Tailscale subnet router，把某个局域网网段发布到 tailnet。这个方案更复杂，需要额外配置路由、审批和防火墙，适合固定家庭服务器或 NAS 场景。

## 不推荐：公网端口映射

不要把 `46881` 控制端口或 `46882` Web 端口直接暴露到公网。即使应用有加入密钥、签名和信任机制，公网裸露仍会增加扫描、爆破、漏洞利用和误配置风险。

## 排查顺序

1. 在系统里确认 Tailscale 显示两台设备在线。
2. 用 `ping 100.x.x.x` 或浏览器打开 `http://100.x.x.x:46882/api/presence` 验证网络是否通。
3. 如果 presence 能打开，但 App 里连接失败，检查加入密钥是否一致。
4. 如果显示待信任但不能控制，到双方设置页的“信任”页签完成信任。
5. 如果传文件失败，先确认远端文件共享已开启，且目标路径在文件库根目录内。
