# 开机自动启动

> 适用于 Lan Control Hub v0.16.1+。

Lan Control Hub 已经支持 **Windows 和 macOS 开机自动启动**，类似 Tailscale 那种登录用户后自动起 App 的体验。Linux 当前不在白名单内。

---

## 在 App 里一键开关

打开 Lan Control Hub → 顶部「设置」→「系统」标签页 → 找到「**开机自动启动**」卡片。

| 状态 | 含义 |
|---|---|
| **未启用** | 开机后不会自动启动 App，需要你手动打开。 |
| **已开启** | 登录 Windows / macOS 用户后，App 会自动启动并直接进入托盘（不出窗口）。 |
| **不支持** | 当前平台不支持（一般是 Linux 或受限 Windows 账户）。 |

点击按钮即可切换；状态变更会同步写到审计日志。

不需要电脑登录密码或管理员权限 — App 用的是当前用户的 `HKCU\Software\Microsoft\Windows\CurrentVersion\Run` 注册表项（Windows）或 `Login Items`（macOS），是当前用户作用域，不需要弹 UAC。

---

## 你需要做什么

**大多数情况：什么都不用做**。在 App 设置里点一下按钮就行。

如果你在企业 / 公司机器上，或者当前用户权限被锁，可能的情况：

| 现象 | 原因 | 怎么解决 |
|---|---|---|
| 按钮显示「不支持」 | App 拒绝在 Linux / 受限 Windows 账户启用 | 换用有管理员权限的账号，或在 Windows 上以管理员运行一次让 App 写入 HKLM 路径 |
| 切换后下次开机没生效 | 组策略 (GPO) 锁了 Run 注册表 | 找 IT 解锁 HKCU\...\Run，或改用任务计划程序开机脚本调用 App |
| 切换后弹 UAC | 极少数情况下 App 误判需要管理员写入 | 取消即可，下次再切换就正常 |

## 跟 Tailscale 自启的关系

Tailscale 自己有自启逻辑（Windows 服务 / macOS LaunchAgent）。两个 App 互相独立：

- 开启 Tailscale 自启：登录系统后 Tailscale 先起，100.x IP 就绪后再启动 Lan Control Hub。
- 不开 Tailscale 自启：Lan Control Hub 启动时如果没 Tailscale 接口，会把 `networkInfo.activeNetwork` 显示为 `lan` 或 `none`，你可以正常加入 / 创建房间。

两者没有冲突。

---

## 验证步骤

1. 在 App 设置里点「开启开机自启」，确认状态变成「已开启」。
2. 完全关机再开机（不是重启）。
3. 登录用户后几秒，托盘会出现 Lan Control Hub 图标，窗口不会自动弹出（用了 `--hidden` 参数）。
4. 双击托盘图标打开主窗口，确认 App 正常联网。

如果没起来：
- Windows：打开「任务管理器 → 启动」一栏，看有没有 `Lan Control Hub` 条目。
- macOS：打开「系统设置 → 通用 → 登录项 → 打开登录时」，看 Lan Control Hub 是否在「允许在后台」列表里。

## 实现说明（开发者）

- IPC: `lch:get-auto-launch` / `lch:set-auto-launch`
- REST: `GET/POST /api/settings/auto-launch`
- 入口：`src/main/index.ts` 的 `getAutoLaunch()` / `setAutoLaunch()`，基于 Electron `app.setLoginItemSettings({openAtLogin, args: ['--hidden']})`。
- 状态字段：`AppStateView.autoLaunch: { enabled, available, reason? }`。
- UI：SetupScreen 顶部展示当前网络；SettingsView → 系统 → 开机自动启动卡片。
- 协议类型：`AutoLaunchInfo` 在 `src/shared/protocol.ts`。

无新依赖，全部用 Electron 内置 API + Electron 在 Windows / macOS 上保证写入正确的注册表 / LoginItems。
