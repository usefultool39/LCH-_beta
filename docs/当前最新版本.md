# 当前最新版本

当前推荐使用版本：`v0.13.0`

`v0.13.0` 是新的房间化体验版本。它保留 v0.12 系列已有的聊天、文件、终端、看屏和远控能力，同时把首次加入流程改成更像“局域网房间大厅”的方式：先扫描附近房间，选中房间，再输入房间密码加入。

## 已具备的工作范围

- 局域网设备发现、房间扫描、加入密钥、设备身份和手动/自动信任。
- 当前房间状态、本机代号、房间代码和退出当前房间。
- 一对一聊天、本地搜索、回复引用、Markdown/代码块、基础 reaction。
- 群组文本会话、小文件发送、群组标题和成员编辑。
- 文件中枢浏览、在线预览、20 GB 级流式下载和上传。
- 远程命令、PTY/xterm 交互终端和 spawn fallback。
- 截图、实时看屏、远程鼠标键盘、远端剪贴板。
- 默认局域网 WebRTC 直连，以及可选 STUN/TURN/relay 配置。
- Local API 和 `lch` CLI，供本机脚本或 Agent 调用。

## 下载包

GitHub Latest Release 应包含：

```text
Lan-Control-Hub-0.13.0-win-x64-portable.exe
Lan-Control-Hub-0.13.0-win-x64-setup.exe
Lan-Control-Hub-0.13.0-mac-x64.zip
Lan-Control-Hub-0.13.0-mac-arm64.zip
SHA256SUMS.txt
```

Windows 日常使用优先选择 setup 安装版；临时测试或不想安装时选择 portable 便携版。macOS 按 CPU 架构选择 `mac-x64` 或 `mac-arm64`。

## 使用前确认

1. 所有电脑尽量升级到同一个 latest release。
2. 第一台电脑创建房间，其他电脑在初始页扫描附近房间。
3. 选中房间后，输入房主电脑设置页显示的房间密码/加入密钥。
4. 新设备出现后双方都要在设置页确认信任；如果开启自动信任，会自动加入信任列表。
5. 如果跨网络使用，优先通过 Tailscale、ZeroTier 或 WireGuard，把对方虚拟 IP 加到网络设置中。
6. 不建议直接做公网端口暴露。

## 当前边界

- Windows 和 macOS 包目前未做付费代码签名，首次打开可能出现来源确认。
- 房间扫描只发现已经打开 Lan Control Hub 且已加入某个房间的设备；没有房间时应创建新房间。
- 群组共享目录大文件、已读回执、云端同步、多显示器选择和协作标注仍属于后续版本。
- 旧设备不认识新群组 metadata 时会降级为普通文本或普通文件消息，不应 crash。
